## 概述

若追蹤代碼無法正常運作，可依下列項目檢查設定路徑與 ID 格式。

## 設定路徑

- **FB Pixel / GTM**：皆在後台 **[設定] > [追蹤]** 區塊中貼入 ID，儲存後即生效。
- 請確認您是在「正確的網站」後台進行設定，若帳號有多個網站，需分別設定。

## 常見錯誤

### ID 格式錯誤

- **Facebook Pixel ID**：應為一組數字（例如 `1234567890123456`），請勿包含空格、引號或前後綴。從 Facebook 事件管理工具中複製「像素編號」即可。
- **GTM Container ID**：格式為 `GTM-XXXXXXX`（例如 `GTM-ABC1234`），請完整貼上，包含 `GTM-` 前綴。從 Google Tag Manager 的「工作區」右上角可找到容器 ID。

### 貼上後沒有數據

- 確認網站已 **發布**，未發布的草稿變更不會出現在線上網站。
- 若使用廣告阻擋或隱私擴充功能，可能無法觸發追蹤，請用無痕模式或另一裝置測試。
- 數據可能有延遲，請等待數小時後再於 Facebook / Google 後台查看。

### 無效或已刪除的 ID

若您更換了 Pixel 或 GTM 容器，請在 Kano [設定] > [追蹤] 中 **更新為新的 ID** 並儲存，舊 ID 將不再紀錄。
